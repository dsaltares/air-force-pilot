Parámetros:
phase Orden de destrucción, a más pequeño, más tarde será destruido el Singleton

Por ejemplo un Singleton Log deberá tener una fase más baja que un Singleton que se encargue de manejar recursos multimedia. Quizás éste último necesite usar el Singleton Log durante su destrucción, si el Log se hubiera destruído, daría problemas graves.
